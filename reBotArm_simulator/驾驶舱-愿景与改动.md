# reBot 驾驶舱 · 愿景与改动说明

> 本目录（`reBotArm_simulator/`）二次开发自社区项目 **[Yang-Ci/Rebot-Arm](https://github.com/Yang-Ci/Rebot-Arm)** 的网页仿真器，在其基础上做了对接、修复与扩展。原作者保留署名，协议沿用 **CERN-OHL-W-2.0**。

## 这是什么
一个 reBot Arm B601-DM 的**网页控制台（"驾驶舱"）**：浏览器里 3D 看机械臂、控制真机、做教学演示。零安装，手机也能开（PWA）。

## 愿景：把整条学习链做成"点一点"的向导
让没基础的人也能跑通机械臂：**装 → 连 → 玩 → 采数据 → 看数据 → 训练 → 看进度 → 部署**，每一步尽量在网页里点按钮完成，并按平台（Windows / WSL / Linux）给**诚实的指引和避坑提示**。

核心理念：很多配置、踩坑经验本来需要老手或 AI 才能跨过，我们把它沉淀进工具，让小白也能自己点过去。

## 本 fork 已做的改动（相对上游）
- **对接本项目维护的 controller**：模型目录与命令接口对齐 `reBotArmController_ROS2`（关节命令 `JointPosVelCmd → /cmd/pos_vel`；夹爪开/合走 `/gripper/open|close` 服务，规避"米 vs 电机 rad"单位错）。
- **一键重力补偿**：接 `/rebotarm/gravity_compensation/start|stop`，按钮态由真机 `arm_status` 驱动。
- **一键遥操（可选端口）**：网页扫串口、选主动/跟随臂端口 → 启停 LeRobot 物理遥操（102 主动 → 601 跟随）。控制端点带**安全闸**（仅本机 127.0.0.1 + POST + 同源，防 LAN/CSRF 远程启动真机）。
- **平台体检条**：自动探测 Windows / WSL / Linux，WSL 提醒 USB 透传约 2Hz。
- **界面**：保持上游原始深色风格不变（视觉留待专业设计师重做，本次只加功能不改皮肤）。

## 跑起来
```bash
cd reBotArm_simulator
npm start         # 纯 Node 零依赖
# 浏览器开 http://localhost:3001
```
纯仿真：拖关节、解 IK、摆预设、示教，**不需要硬件**。真机控制见控制台「ROS2 桥接」区（需 ROS2 + rosbridge）。

## 平台说明
| 平台 | 仿真 | 真机控制 |
|---|---|---|
| **原生 Linux** | ✅ | ✅ 全速，实时遥操/重力补偿无瓶颈 |
| **Windows 原生** | ✅ | 🟡 走 COM 口，实时遥操未实测 |
| **WSL2** | ✅ | ⚠️ USB 经 usbipd 透传约 **2Hz**，仅够点动/调试，实时遥操请用原生 Linux |
